# Анализ нормативки: заявка, документы, проверки (ЛС и МИ)

Извлечено из корпуса `ЛС/` и `МИ/` (распарсено в `md_извлечено/`, структурировано в БД `reference_sections`).
Ключевые источники:
- **ЛС:** Решение Совета ЕЭК № 78 (Правила регистрации и экспертизы ЛС) — `ls-2`; ОХЛП/инструкция/маркировка — Решение № 88 (`ls-10`); формы заявлений и перечень документов — приложения к Решению 78.
- **МИ:** Решение Совета ЕЭК № 46 (Правила регистрации и экспертизы МИ) — `mi-2`; Общие требования безопасности — Решение № 27 (`mi-5`); классификация по риску — Решение № 173 (`mi-11`); номенклатура — Решение № 177 (`mi-12`); СМК — Решение № 106 (`mi-4`); техиспытания — № 28 (`mi-6`); биологическое действие — № 38 (`mi-7/10`); клинические испытания — № 29 (`mi-8`).

# 1. ПОДАЧА ЗАЯВКИ И ПРОЦЕДУРА

## 1.1 ЛС — процедуры (Решение 78, разделы I–XV)

| Процедура | Раздел Правил | Суть |
|---|---|---|
| **Регистрация** | III (общие принципы), V (взаимное признание), VI (децентрализованная) | заявление + досье → начальная экспертиза (комплектность) → специализированная экспертиза → инспекция GMP при необходимости → экспертное заключение |
| **Перерегистрация** (подтверждение регистрации) | **VIII** | подтверждение через 5 лет; обновлённое досье, PSUR/отчёт по безопасности |
| **Внесение изменений** | **IX** | изменения в регистрационное досье зарегистрированного препарата (типы IA/IB/II по классификации) |
| Приведение в соответствие с правом Союза | XIII | перевод нац. досье в формат ЕАЭС |
| Приостановка/отзыв/ограничение | X | пострегистрационные меры |

**Этапы (для MVP важны начальные):**
1. Подача заявления + досье + оплата (+ образцы/стандарты при необходимости)
2. **Начальная экспертиза = комплектность + правильность оформления** ← зона MVP
3. Специализированная экспертиза (качество/безопасность/эффективность)
4. Лабораторные испытания при необходимости
5. Замечания эксперта → ответ заявителя
6. Экспертное заключение

### Форма ЗАЯВЛЕНИЯ ЛС (3 варианта — приложение к Решению 78)
- **I. ЗАЯВЛЕНИЕ о регистрации** (стр. 3343 md)
- **II. ЗАЯВЛЕНИЕ** — подтверждение регистрации/перерегистрация (стр. 3657)
- **III. ЗАЯВЛЕНИЕ** — внесение изменений (стр. 3968)

**Общие поля заголовка (всегда):**
- Торговые наименования лекарственного препарата
- Активные фармацевтические субстанции (АФС)
- Дозировки/концентрация и лекарственная форма
- Форма выпуска
- Держатель регистрационного удостоверения
- Заявитель
- Представитель заявителя
- Гарантия достоверности (подтверждение заявителя)

**1. Общие пункты — выбор процедуры (1.1), от него зависят поля:**
- 1.1.1 **по процедуре взаимного признания** → референтное государство, торг. наименование в реф. государстве, дата регистрации, № РУ, копия РУ, идентификационный № заявления, государства признания + их торг. наименования/даты/№ РУ
- 1.1.2 **по децентрализованной процедуре** → идентификационный № заявления, референтное государство, государства признания для подачи
- 1.1.3 **приведение в соответствие** → реф. государство, № РУ, идентификационный № заявления на приведение

**Тип препарата (чекбокс) — ГЛАВНЫЙ условный драйвер полей и документов:**
| Тип | Появляющиеся поля |
|---|---|
| **Оригинальный** | новая АФС / известная АФС |
| **Воспроизведённый (generic)** | однокомпонентный/многокомпонентный; оригинальный препарат (наименование, дозировка, ЛФ, держатель РУ, дата, №); **референтный препарат, использованный в исследованиях эквивалентности**; обоснование выбора референтного; рекомендации Экспертного комитета |
| **Биоподобный (биоаналог)** | оригинальный биологический препарат; референтный биологический препарат; различия по сравнению с референтным |
| **Гибридный** | оригинальный препарат + отличия |
| (далее в форме) травяной, гомеопатический, well-established use, орфанный, radiopharm, из крови — со своими условиями |

> **Вывод для приложения:** поле «тип препарата» (`param-product-type`) и «процедура» (`param-procedure`) — главные условные триггеры. От них зависят: какие поля показывать (референтный препарат — только для generic/biosimilar/hybrid), и какие документы обязательны.

## 1.2 МИ — процедуры (Решение 46)

| Процедура | Где | Суть |
|---|---|---|
| **Регистрация** | раздел II (п.21–34) | выбор реф. государства + ≥1 государства признания; заявления (Приложения 2,3) + досье (Приложение 4) → проверка полноты (7 раб. дней) → экспертиза → инспекция СМК при необходимости → экспертное заключение → согласование |
| **Перерегистрация / согласование на зарегистрированное** | разделы далее | подтверждение |
| **Внесение изменений** (в т.ч. уведомительный порядок) | отдельный раздел | изменения в досье |
| Приостановление/отмена РУ | заключит. разделы | — |

**ГЛАВНЫЙ условный драйвер МИ — `класс потенциального риска`** (1, 2а, 2б, 3 по Решению № 173) и **вид по номенклатуре** (Решение № 177). От класса риска зависят: объём досье, требования к СМК (Решение 106), необходимость клинических испытаний, тип доказательств.

**Заявления МИ (Решение 46):**
- Приложение № 2 — заявление о проведении **экспертизы**
- Приложение № 3 — заявление о проведении **регистрации**

> **Вывод для приложения:** для МИ ключевые поля-триггеры — `класс потенциального риска` и `вид по номенклатуре` (аналог product-type для ЛС).

## 1.3 Какие поля влияют на проверку экспертом
- **ЛС:** `процедура` (регистрация/перерег/изменения) → меняет состав досье и набор проверок; `тип препарата` → generic требует эквивалентность/референтный препарат, biosimilar — сравнимость, оригинальный — полное досье; `новая/известная АФС` → мастер-файл АФС; наличие исследований эквивалентности.
- **МИ:** `класс риска` → глубина экспертизы и СМК; `вид по номенклатуре`; наличие ЛС в составе, стерильность, ПО, измерительная функция → доп. проверки (см. п.27).

# 2. КАКИЕ ДОКУМЕНТЫ ГРУЗИТЬ И КОГДА

## 2.1 ЛС — регистрационное досье (ОТД/CTD, 5 модулей; Решение 78, прил. № 1)

> ОТД состоит из 5 модулей. Модуль 1 специфичен для государств-членов; модули 2–5 общие.

- **Модуль 1. Административные сведения**
  - 1.1 Содержание
  - 1.2 Общая документация: заявление, ОХЛП, инструкция (ИМП/ЛВ), макеты упаковок, сведения об экспертах, фармаконадзор (мастер-файл системы фармаконадзора, RMP), документы по GMP/CPP, оплата
- **Модуль 2. Резюме ОТД** (2.1 содержание, 2.2 введение, 2.3 общее резюме по качеству, 2.4 обзор доклинических, 2.5 обзор клинических, 2.6 резюме доклинических, 2.7 резюме клинических)
- **Модуль 3. Качество** (3.1 содержание, 3.2 основные сведения: **3.2.S — АФС**, **3.2.P — препарат**, 3.2.A — приложения, 3.2.R — региональная информация; 3.3 литература)
- **Модуль 4. Отчёты о доклинических (неклинических) исследованиях** (4.1 содержание, 4.2 отчёты, 4.3 литература)
- **Модуль 5. Отчёты о клинических исследованиях** (5.1 содержание, 5.2 …, отчёты по биоэквивалентности — для generic)

**Условность состава (Решение 78, раздел «Специальные требования к документам отдельных видов», стр. 2609):**
| Тип препарата | Особенности досье |
|---|---|
| Оригинальный | полные модули 4 и 5 (собственные доклинические + клинические) |
| **Воспроизведённый (generic)** | модуль 4/5 сокращены; **обязательны отчёты по биоэквивалентности** (модуль 5) |
| Биоподобный | сравнительные исследования с референтным биопрепаратом |
| С известной АФС | **мастер-файл АФС** (Решение 78, прил. — процедура мастер-файла АФС) |
| Высокотехнологичные (ATMP) | раздел IV спец. требований |
| Травяные/гомеопатические | упрощённые модули по монографиям |

Полный перечень документов в модулях — Решение 78, «I. Перечень документов в модулях регистрационного досье» (стр. 4314 md).

## 2.2 МИ — документы (Решение 46, п.22 + Приложение № 4)
Заявитель представляет:
- **а)** регистрационное досье = заявления о регистрации и экспертизе + документы по перечню **Приложения № 4**
- **б)** копии документов об оплате

Состав досье МИ зависит от **класса риска** и включает (по п.27 экспертизы): доказательства соответствия Общим требованиям (Реш. 27), протоколы технических испытаний (Реш. 28), отчёт по биологическому действию (Реш. 38), отчёт о клиническом доказательстве (Реш. 29), отчёт по анализу рисков, отчёт о стабильности (срок хранения), данные о валидации ПО (если есть ПО), данные о стерилизации (если стерильное), эксплуатационный документ/инструкция, маркировка, данные о СМК (Реш. 106), сведения о производственных площадках.

# 3. ТРЕБОВАНИЯ К РАЗДЕЛАМ / ЧТО ПРОВЕРЯЕТ ЭКСПЕРТ

## 3.1 ЛС — что проверяется (Решение 78)
- **Начальная экспертиза:** комплектность модулей 1–5 по типу процедуры и препарата; правильность оформления; соответствие формы заявления; наличие ОХЛП/инструкции/макетов; оплата; GMP/CPP.
- **Специальные требования к модулям** (раздел II спец. требований, стр. 2325): по каждому модулю — обязательные подразделы и их содержание.
- **Специальные требования по видам препаратов** (раздел III, стр. 2609): generic → биоэквивалентность; biosimilar → сравнимость; ATMP → раздел IV.
- **Расхождения между документами** (ключевая авто-проверка MVP): срок годности, условия хранения, состав, дозировка, адреса/производители — должны совпадать между заявлением, ОХЛП, инструкцией, макетами, модулем 3.
- **ОХЛП/инструкция/маркировка** — отдельные требования к содержанию и структуре (Решение № 88, `ls-10`; в БД 250 требований — самый детальный источник по пунктам).

## 3.2 МИ — что проверяет эксперт (Решение 46, п.27 — полный чек-лист)
Экспертиза включает (если применимо):
- **а)** правильность отнесения к медицинским изделиям
- **б)** соответствие заявленного **класса потенциального риска** (Реш. 173)
- **в)** правильность отнесения к виду по **номенклатуре** (Реш. 177)
- **г)** соответствие моделей/марок критериям одного РУ (п.19)
- **д)** анализ доказательств соответствия **Общим требованиям** (Реш. 27):
  - протоколы технических испытаний (Реш. 28)
  - оценка биологического действия (Реш. 38) + отчёт по биобезопасности (материалы животного/человеческого происхождения)
  - отчёт о клиническом доказательстве эффективности и безопасности
  - клинические/клинико-лабораторные испытания (Реш. 29)
  - анализ ЛС в составе изделия (если есть)
  - **стерилизация** — методы, контроль остатков (если стерильное)
  - **валидность ПО** — верификация/валидация, защита от несанкц. доступа (если есть ПО)
  - **отчёт о стабильности** — обоснование срока хранения
  - **отчёт по анализу рисков** — полнота идентифицированных опасностей
  - соответствие эксплуатационного документа/инструкции Общим требованиям
  - соответствие **маркировки** Общим требованиям
  - технические характеристики
  - испытания для утверждения типа средств измерений (если измерительная функция)
- **е)** отчёты об инспектировании производства
- **ж)** данные о сертификации **СМК** (Реш. 106)
- **з)** обоснованность производственных площадок
- **и)** данные о разработке и производстве (схемы процессов)
- **к)** информация о маркетинге
- **л)** сведения о неблагоприятных событиях/отзывах
- **м)** план сбора данных по безопасности на постпродажном этапе

# 4. Карта «регуляция → приложение»

| Регуляторный элемент | Где в приложении |
|---|---|
| Тип препарата ЛС / класс риска МИ | `param-product-type` / поле класса риска — условные триггеры (`requiredWhenExpression`) |
| Процедура (рег/перерег/изменения) | `param-procedure` — фильтр правил и проверок |
| Модули CTD 1–5 / разделы | `lib/dossier/section-definitions.ts`, `document_requirement_rules` (175) |
| Перечень документов по условиям | `lib/rules/engine.ts` → `getRequiredDocuments()` |
| Требования по пунктам НПА | `reference_*` + admin «Требования» (302) |
| Проверки внутри документа | `lib/checks/registry.ts` (26 проверок) + `npa-gemma-check` (LLM по требованиям) |

> Самый детальный источник требований по пунктам — **Решение 88** (ОХЛП, `ls-10`, 250 разделов) и **Решение 78** спец. требования к модулям. Для МИ — **п.27 Решения 46** как готовый чек-лист эксперта.
