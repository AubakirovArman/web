# Пробелы по НПА из `info.md` для MVP

В `info.md` выделен ядровый набор НПА, на которые система должна ссылаться в MVP. Ниже — что из них уже реализовано в коде, а что ещё не сделано, с привязкой к типу процедуры.

|---|---|---|---|
| 10 | 4 | 4 | 2 |

**Тип процедуры:**
- **Р** = Регистрация
- **П** = Перерегистрация
- **И** = Внесение изменений
- **Все** = сквозное требование для любой процедуры

## 1. Приказ ҚР ДСМ-10 — «Правила проведения экспертизы ЛС или МИ»

| | |
|---|---|
| **Файл** | `ЛС/22. Приказ МЗ РК от 27.01.2021 г. № ҚР ДСМ-10.docx` |
| **Тип процедуры** | Все (форма заявления + перечень документов + изменения) |
| **Статус** | ⚠️ Частично |

### Что уже сделано
- NPA заведён в `lib/data/seed.ts` (`npa-8`).
- В `lib/checks/engine.ts` на него ссылаются проверки:
  - комплектности полей заявления (Application completeness);
  - форматов файлов;
  - совпадения производителя / адреса площадки;
  - сравнения RU/KZ-версий ОХЛП/инструкции;
  - наличия ПУР для биологических/биоаналогичных;
  - структуры Модуля 3;
  - стерильных препаратов.

### Что не сделано
- Приложение 1 (форма заявления): в форме заявителя не все поля из ДСМ-10 — нет упаковки, состава, держателя РУ, фармаконадзора, признаков «есть ГМО/вещества животного или человеческого происхождения» и др.
- Приложение 2 (перечень документов для производителей РК): не полностью разложен по типам препарата.
- Приложение 4 (материалы досье в зависимости от вида ЛС): не используется для формирования чеклиста под оригинальный, гибридный, биологический, биоаналогичный, вакцину и т. д.
- Сценарии перерегистрации и внесения изменений по ДСМ-10 не реализованы.

## 2. Решение №88 — «Требования к инструкции и ОХЛП»

| | |
|---|---|
| **Файл** | `ЛС/10. Решение № 88 от 3 ноября 2016 года.docx` |
| **Тип процедуры** | Все (ОХЛП, инструкция, макет) |
| **Статус** | ⚠️ Частично |

### Что уже сделано
- NPA заведён (`npa-5`).
- Проверки в `engine.ts`:
  - срок годности в ОХЛП vs инструкцией / НДК / стабильностью;
  - условия хранения в ОХЛП vs инструкцией / макетом;
  - сравнение объёма RU/KZ-версий;
  - обязательные элементы в тексте маркировки (торговое наименование, МНН, дозировка).

### Что не сделано
- Требования к шрифту (Times New Roman, 12 пт, цвет, стиль) — не проверяются.
- Чёрный перевернутый треугольник / дополнительный мониторинг безопасности — не проверяется.
- Структура разделов ОХЛП и инструкции (наличие обязательных разделов) — не проверяется.
- Удобочитаемость / формат листка-вкладыша — не проверяется.

## 3. Решение №85 — «Правила проведения исследований биоэквивалентности»

| | |
|---|---|
| **Файл** | `ЛС/4. Решение № 85 от 3 ноября 2016 года.docx` |
| **Тип процедуры** | Регистрация (generic) |
| **Статус** | ⚠️ Частично |

### Что уже сделано
- NPA заведён (`npa-3`).
- Правило `rule-generic-bioequivalence` требует отчёт БЭ или биовейвер.
- Проверки в `engine.ts`:
  - наличие отчёта/обоснования для generic;
  - дозировка в отчёте БЭ vs заявление;
  - указан референтный препарат;
  - вывод отчёта подтверждающий.

### Что не сделано
- Проверка **лекарственной формы** в отчёте БЭ (в `info.md` упоминается `dosage_form_consistency`).
- Проверка биовейвера: если загружено обоснование отсутствия БЭ, оно не анализируется на корректность.
- Нет сравнения референтного препарата из отчёта БЭ с референтом в заявке/ОХЛП.

## 4. Решение №77 — «Правила надлежащей производственной практики (GMP)»

| | |
|---|---|
| **Файл** | `ЛС/7. Решение № 77 от 3 ноября 2016 года.docx` |
| **Тип процедуры** | Все / GMP |
| **Статус** | ⚠️ Частично |

### Что уже сделано
- NPA заведён (`npa-4`).
- Правило `rule-sterile` для стерильных препаратов.
- Проверки GMP:
  - срок действия;
  - предупреждение об истечении через 6 месяцев;
  - совпадение адреса площадки с заявлением;
  - совпадение производителя с заявлением.

### Что не сделано
- Проверка, что GMP покрывает конкретную **лекарственную форму** / производственную линию.
- Проверка номера/области действия GMP-сертификата.
- Нет связи с Приказом ДСМ-9 (инспекции по НФП).
