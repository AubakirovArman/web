# Логика проверок загруженных документов по типу процедуры

Цель: помочь эксперту быстро увидеть риски и несоответствия в заявке, независимо от того, это регистрация, перерегистрация или внесение изменений.

Общий принцип:
- Сначала проверяем **комплектность** (какие документы должны быть, какие загружены).
- Потом **форматы и читаемость**.
- Потом **содержание каждого документа**.
- Потом **согласованность данных между документами**.
- В конце — **процедурно-специфичные проверки**.

### 1.3. Проверки по документам

#### Заявление (`doc-application`)
- Извлечь: торговое наименование, МНН, дозировка, лекарственная форма, производитель, адрес, заявитель, держатель РУ.
- Проверить, что обязательные поля заполнены.
- Сверить с параметрами, указанными заявителем в wizard.

#### GMP (`doc-gmp`)
- Срок действия не истёк.
- До истечения не менее 6 месяцев (warning, если ≤ 6 мес.).
- Производитель совпадает с заявлением.
- Адрес площадки совпадает с заявлением.
- GMP покрывает заявленную лекарственную форму (если удалось извлечь scope).

#### CPP (`doc-cpp`)
- Срок действия не истёк.
- Дата выдачи не старше 5 лет (warning).
- Страна выдачи совпадает со страной производства.

#### ОХЛП (`doc-spc-ru`, `doc-spc-kz`)
- Есть RU и KZ версии.
- Объём RU/KZ отличается не более чем на 40 %.
- Содержит обязательные поля: торговое наименование, МНН, дозировка, срок годности, условия хранения.
- Шрифт/оформление — проверяем для DOCX (Times New Roman, 12 пт), для PDF — informational warning.

#### Инструкция (`doc-instruction-ru`, `doc-instruction-kz`)
- Есть RU и KZ версии.
- Содержит обязательные разделы (состав, показания, дозировка, побочные действия и т. д.) — упрощённо.
- Объём RU/KZ согласован.

#### Макеты упаковки (`doc-mockup`)
- Формат JPEG/PNG.
- Извлечены: торговое наименование, дозировка, срок годности, условия хранения.

#### Модуль 3 (`doc-module3`)
- Подтверждены разделы: спецификация, валидация методов, данные стабильности.
- Для стерильных препаратов — валидация стерильности.

#### Отчёт биоэквивалентности (`doc-bioequivalence-report`) — только generic
- Указан референтный препарат.
- Дозировка совпадает с заявлением.
- Лекарственная форма совпадает с заявлением.
- Вывод подтверждающий.

| GMP-сертификат | `doc-gmp` | critical | |

| Мастер-файл фармаконадзора | `doc-pharmacovigilance-master` | warning | |

### 2.2. Процедурно-специфичные проверки

#### Регистрационное удостоверение (`doc-registration-certificate`)
- Срок действия не истёк.
- Номер РУ совпадает с указанным в заявлении (`param-registration-number`).
- Торговое наименование совпадает с заявлением.
- МНН совпадает с заявлением.
- Дозировка совпадает с заявлением.

#### Обновлённое досье
- Модули 1–2 присутствуют.
- Модуль 3 — не обязателен, но если качество вызывает сомнения, запрашивается (informational warning, если отсутствует).

#### Пострегистрационные данные (`doc-post-marketing-data`)
- Отчёт не старше 5 лет.
- Содержит сводку по безопасности и эффективности.

#### GMP
- Те же проверки, что при регистрации.
- Если срок GMP истекает раньше, чем срок РУ — warning.

### 2.3. Сквозные проверки

- Те же, что при регистрации, плюс:
- Сравнение текущей ОХЛП/инструкции с предыдущей редакцией (если есть) — выявить незаявленные изменения.

### 3.3. Процедурно-специфичные проверки

#### Описание изменений (`doc-variation-description`)
- Извлечь: класс изменения (IA / IB / II), область изменения, старое значение, новое значение, описание, затронутые документы.
- Класс изменения совпадает с выбранным в заявке (`param-variation-class`).
- Область изменения совпадает с выбранной в заявке (`param-variation-area`).
- Старое и новое значение указаны.

#### Обоснование изменений (`doc-variation-justification`)
- Текст обоснования присутствует.
- Для класса II — обязательно (critical).
- Для класса IB — обоснование должно быть (serious).
- Для класса IA — может быть кратким.

#### Сравнительная таблица (`doc-variation-comparison`)
- Указаны старое и новое значение.
- Указан раздел документа.
- Перечислены затронутые документы.

#### Действующая vs проектная ОХЛП
- Торговое наименование не должно меняться (если меняется — critical, это не variation, а новая регистрация).
- МНН не должно меняться.
- Дозировка не должна меняться без обоснования.
- Изменённое значение должно быть явно выделено в проекте ОХЛП.
- Срок годности / условия хранения — если изменены, проверить наличие данных стабильности.

#### Маркировка
- Если область изменения = `labeling` или `spc`:
  - текст маркировки загружен;
  - макет упаковки загружен;
  - новые данные отображены в макете.

#### GMP / производство
- Если область изменения = `manufacturing`:
  - GMP загружен;
  - CPP загружен;
  - производитель/адрес в новых документах согласован.

#### Качество
- Если область изменения = `quality`:
  - НДК загружен;
  - Модуль 3 загружен;
  - стабильность загружена.

### 3.4. Сквозные проверки

| Что сравниваем | Источники | Severity |
|---|---|---|
| Старое значение | `doc-variation-description`, `doc-variation-comparison`, `doc-current-spc-ru` | serious |
| Новое значение | `doc-variation-description`, `doc-variation-comparison`, `doc-updated-spc-ru` | serious |
| Класс изменения | `param-variation-class`, `doc-variation-description` | serious |
| Область изменения | `param-variation-area`, `doc-variation-description` | warning |
| Согласованность RU/KZ проектных ОХЛП | `doc-updated-spc-ru`, `doc-updated-spc-kz` | warning |

## 4. Общие проверки для всех процедур

- Формат файла соответствует допустимому для типа документа.
- Файл не пустой и не битый.
- Нет дублей документов одного типа.
- Язык документа определён (RU/KZ).
- Документ распознан/имеет текстовый слой (OCR warning, если нет).

---

## 5. Категории замечаний и severity

| Severity | Когда выдавать | Пример |
|---|---|---|
| **critical** | Нарушение, при котором заявку нельзя принимать | Отсутствует GMP, просрочен РУ, нет обязательного документа |
| **serious** | Существенное несоответствие, требует уточнения | Расхождение дозировки, отсутствие обоснования изменений |
| **warning** | Риск/подозрение, требует проверки экспертом | Срок GMP истекает через 6 месяцев, объём перевода отличается > 40 % |
| **info** | Информация, не требующая действия | Модуль 3 не обязателен при перерегистрации, но рекомендуется |
