# A12. Ограничения и технический долг MVP

Сводка по всем артефактам части A. Приоритет: 🔴 критично · 🟡 средне · 🟢 мелочь.

## Функциональные пробелы
- 🔴 **Покрыта только ЛС/регистрация.** `document_requirement_rules` = 174, все `LS/registration`. Нет МИ; нет перерегистрации; нет внесения изменений (хотя поля и профили форм для них есть, и есть словарь `ls_variation_checklist_rules` 437 kB — не задействован движком документов).
- 🔴 **Асимметрия движка ЛС vs МИ:** для ЛС — богатый admin-движок с текстовыми триггерами; для МИ — только seed-правила (бедные).
- 🟡 **Двойная логика проверок клиент/сервер:** `runPreCheck` исполняется и в wizard (клиент), и в expert (сервер `/check`). Риск расхождения.
- 🟡 **Часть проверок реестра (15–26)** — статус реализации неясен (возможны заглушки); сверить runner'ы.

## Данные и архитектура
- 🔴 **Слабая нормализация:** заявки и конфиг — в JSONB (`runtime_applications.data`, `admin_runtime_config.data`). Файлы/находки/чеклист/результаты LLM не вынесены → 1 заявка ≈ 1.2 МБ, тяжёлые чтения, нет аналитики.
- 🟡 **Reference-БД и runtime-БД совмещены** в одной физической базе `ndda_reference_kb`.
- 🟡 **Несколько слоёв правил ЛС** (`*_raw` → нормализованные → таблица) — неясен каноничный источник и пайплайн.
- 🟢 **3 backup-таблицы** `document_requirement_rules_backup_*` — мусор в схеме.

## Справочник
- 🟡 **Две параллельные системы:** experiment (38, с анализом, в UI) vs `reference_documents` (68, полнотекст, не в UI). Дублирование.
- 🟢 **`/api/reference`, `/api/reference/[id]`** — legacy без потребителей (но `db.ts` нужен npa-gemma-preview).
- 🟡 `intelligence` генерируется офлайн — нет инкрементального обновления при правке НПА.

## LLM/качество
- 🔴 **Внешний vLLM — единая точка отказа**, fallback'и отключены (без сервисов извлечение/проверка падают).
- 🟡 Нет кэширования результатов LLM по (файл, требование) — повторные прогоны дороги.
- 🟡 Качество проверки зависит от OCR/извлечения; есть отчёт `docs/npa-gemma-quality-report.md`.

## UX / интерфейс (из прошлого аудита)
- 🔴 **Нет мобильной навигации** (шапка `hidden md:flex`).
- 🟡 **Нет metadata** (title/description) в layout.
- 🟡 **`window.confirm`** при удалении заявки (вместо AlertDialog).
- 🟡 **Формы не используют установленные** react-hook-form/zod (ручной useState, ad-hoc валидация).
- 🟡 Нет активного пункта в верхней навигации; нет хлебных крошек; широкие таблицы → горизонтальный скролл.
- 🟢 `prefers-reduced-motion` не учитывается.

## Безопасность/эксплуатация
- 🟡 **Нет аутентификации/разграничения ролей** — любой видит зоны заявителя/эксперта/админа.
- 🟢 Парсеры копят job-файлы в `.runtime/` без очистки.
- 🟢 Ручной деплой (по решению владельца — автодеплой не нужен).

## Тесты
- 🟡 Нет юнит-тестов движков; LLM-пайплайн не тестируется; перерег/изменения и МИ-документы не покрыты на уровне правил.

- ✅ Микросервисы извлечения/LLM — переносимы.

- ✅ 174 нормализованных правила ЛС с привязкой к полям и пунктам НПА — ценная база.
