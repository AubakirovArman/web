# A13. Тесты

Playwright e2e: **9 spec-файлов** (`web/e2e/*.spec.ts`). Юнит-тестов движков (правила/проверки) нет.

## Покрытие по сценариям
| Spec | Сценарии |
|---|---|
| `navigation.spec.ts` | основные маршруты рендерятся; навигация; **тема (light/dark) не ломает приложение** |
| `wizard.spec.ts` | инициализация, сохранение черновика, переходы между секциями с предупреждениями; секция документов показывает карточки загрузки; загрузчик отклоняет неверный формат |
| `application-e2e.spec.ts` | полный цикл: создание ЛС-регистрации → проверка → submit → открытие экспертом; submit-валидация блокирует неполный пакет |
| `expert-scenarios.spec.ts` | ideal LS проходит матрицу; неполная ЛС блокируется; missing GMP виден как failed-строка; expired CPP → находка; field mismatch + DOCX format → находки уровня содержимого; generic принимает waiver биоэквивалентности как альтернативу |
| `expert-admin.spec.ts` | эксперт инспектирует, перезапускает проверки, меняет статус; принимает/отклоняет находки; формирует запрос заявителю |
| `admin-checks-gemma.spec.ts` | admin: реестр проверок (поля/форматы/карта проверок), экспорт пакета правил, метаданные правил, источник правила → диалог доказательств; **маппинг Gemma-preview из Решения 88 в локальный тип ОХЛП** |
| `reference.spec.ts` | страница справочника рендерит БЗ ЛС/МИ с поиском; умный справочник содержит все обработанные документы MVP |
| `subject-matter.spec.ts` | предметные проверки (МИ класс III требует биологические и клинические документы; контролы процедуры/домена показывают поля вариаций МИ) |
| `legacy-storage.spec.ts` | устаревшее browser-хранилище не «обнуляет» wizard |

## Что хорошо покрыто
- Основные пользовательские пути (wizard → expert), демо-сценарии (ideal + 4 негативных), админ-инструменты, справочник, тема.

## Пробелы покрытия (для C)
- Нет тестов движка правил/проверок на уровне юнитов (только через UI).
- Нет тестов LLM-пайплайна (npa-gemma-check) — внешняя зависимость.
- МИ покрыт точечно (класс III, вариации) — но правил документов МИ в БД нет; тесты опираются на seed-логику.
- Нет тестов перерегистрации/изменений ЛС как полноценных процедур.
- Нет нагрузочных/перф-тестов (хотя в сессии были перф-правки).
- Запуск: `npm run test:e2e` (требует поднятого приложения и сервисов).
