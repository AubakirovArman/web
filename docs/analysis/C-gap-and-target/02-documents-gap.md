## Качество существующих правил (проверить)
- Полнота: сверить 174 правила против «Перечня документов в модулях» Решения 78 (стр. 4314 md) — нет ли пропущенных позиций модуля 3 (3.2.S.*/3.2.P.* подразделы).
- Корректность условий: 94 `conditional_required` — проверить, что триггеры (`requiredWhenExpression`) соответствуют НПА (выборочно).
- 6 правил `needs_new_param` — требуют доопределить поля (напр. `param-license-or-patent-active`).
- Источник: правила нормализованы (в т.ч. ИИ) → нужна выборочная сверка с первоисточником.

## Рекомендации (приоритет)
1. **ЛС/изменения** — задействовать существующий `ls_variation_checklist_rules` (быстрый выигрыш, данные уже есть).
2. **ЛС/перерегистрация** — небольшой пакет (модули 1–2 + PSUR/ПУР).
3. **МИ/регистрация** — построить набор правил из Приложения № 4 Решения 46 с драйвером «класс риска» (крупнейшая работа).
4. Сверить полноту модуля 3 ЛС (самый детальный, 3.2.S/3.2.P).
5. Закрыть 6 `needs_new_param` (доопределить поля формы).
6. Единая схема `document_requirement_rules` с scope = {object-type × procedure} (расширить enum scope).
