# B7. Методология экспертизы (что и как проверяет эксперт)

## ЛС — экспертиза (Решение 78, разд. IV + спец. требования)
**Два уровня:**
1. **Начальная экспертиза (комплектность и оформление)** — зона MVP:
   - наличие всех обязательных документов модулей 1–5 по типу процедуры и препарата;
   - правильность оформления (язык, структура, формы);
   - корректность заявления (соответствие формы, заполненность);
   - наличие ОХЛП/инструкции/макетов; оплата; CPP/GMP.
2. **Специализированная экспертиза (качество/безопасность/эффективность):**
   - **Качество (модуль 3):** соответствие Фармакопее ЕАЭС (№100), GMP (№77); 3.2.S (АФС) и 3.2.P (препарат); стабильность → срок годности/хранение; контроль качества.
   - **Безопасность/эффективность (модули 4/5):** доклиника (GLP №81), клиника (GCP №79); для generic — **биоэквивалентность** (или обоснованный waiver); для биоаналога — сравнимость.
   - **Согласованность информации:** ОХЛП ↔ инструкция ↔ макеты ↔ модуль 3 (срок годности, хранение, состав, дозировка, показания).
   - **Фармаконадзор:** мастер-файл системы, ПУР/RMP, контактное лицо, доп. мониторинг (чёрный треугольник).
3. **Замечания → запрос заявителю** (однократно/итеративно) → ответ → экспертный отчёт по оценке → решение.

**Основания для замечаний/отказа:** неполный пакет; несоответствие требованиям к разделам (Реш. 88/78); расхождения между документами; недоказанные качество/безопасность/эффективность; несоответствие GMP.

## МИ — экспертиза (Решение 46, п.27 — готовый чек-лист а–м)
Экспертиза включает (если применимо):
- **а)** правильность отнесения к медицинским изделиям
- **б)** соответствие заявленного **класса потенциального риска** (Реш. 173)
- **в)** правильность отнесения к виду по **номенклатуре** (Реш. 177)
- **г)** соответствие моделей/марок критериям одного РУ (п.19)
- **д)** анализ доказательств соответствия **Общим требованиям** (Реш. 27):
  - протоколы **технических испытаний** (Реш. 28)
  - оценка **биологического действия** (Реш. 38) + биобезопасность материалов
  - **клиническое доказательство** эффективности/безопасности; клинические/клинико-лабораторные испытания (Реш. 29)
  - анализ **ЛС в составе** изделия (если есть)
  - **стерилизация** — методы, контроль остатков (если стерильное)
  - **валидность ПО** — верификация/валидация, защита (если есть ПО)
  - **отчёт о стабильности** — обоснование срока хранения
  - **отчёт по анализу рисков** — полнота опасностей
  - соответствие **эксплуатационного документа/инструкции** Общим требованиям
  - соответствие **маркировки** Общим требованиям
  - технические характеристики
  - испытания для **утверждения типа средств измерений** (если измерит. функция)
- **е)** отчёты об **инспектировании производства**
- **ж)** данные о сертификации **СМК** (Реш. 106)
- **з)** обоснованность производственных площадок
- **и)** данные о разработке/производстве (схемы процессов)
- **к)** информация о маркетинге
- **л)** сведения о неблагоприятных событиях/отзывах
- **м)** план постпродажного мониторинга безопасности

**Сроки/решения:** проверка полноты 7 раб. дней; заключение 45 раб. дней (или инспекция СМК ≤90); согласование государствами признания. Основания отказа — п.34.

## Сопоставление с автоматизацией (для C3)
- **Хорошо автоматизируется:** комплектность, форматы, OCR, расхождения значений, наличие разделов/полей, сроки сертификатов, согласованность ОХЛП/инструкции/макетов.
- **Частично (LLM):** смысловые требования (npa-gemma-check) — соответствие текста требованию пункта НПА.
- **Остаётся эксперту:** научная оценка качества/безопасности/эффективности, оценка «польза–риск», валидность доказательств, классификация риска МИ.
- MVP реализует первый уровень (начальная экспертиза) для ЛС; методология МИ (п.27) — готовый чек-лист, в MVP не реализована.
