# B4. Формы заявлений → поля и условная логика

Источник: Решение 78 (приложение с формами заявлений ЛС — 3 формы) и Решение 46 (Приложения № 2, 3 — формы МИ).

## ЛС — 3 формы заявления (Решение 78)
- **Форма I** — о регистрации (приведении в соответствие)
- **Форма II** — о подтверждении регистрации (перерегистрации)
- **Форма III** — о внесении изменений в регистрационное досье

### Общий заголовок (все формы)
Дата поступления · **Торговые наименования** препарата · **Активные фармацевтические субстанции (АФС)** · **Дозировки/концентрация и лекарственная форма** · **Форма выпуска** · **Держатель РУ** · **Заявитель** · **Представитель заявителя** · гарантия достоверности.

### Раздел 1. Общие пункты — выбор процедуры (1.1) → условные поля
- **1.1.1 взаимное признание:** референтное государство, торг. наименование в реф. гос., дата регистрации, № РУ, копия РУ, идентификационный № заявления, государства признания (+ их торг. наименования/даты/№ РУ), иные государства.
- **1.1.2 децентрализованная:** идентификационный № заявления, реф. государство, торг. наименование, государства признания для подачи.
- **1.1.3 приведение в соответствие:** реф. государство, № РУ, идентификационный № заявления на приведение, государства, где уже зарегистрирован.

### Тип препарата (чекбокс) → главный условный блок полей
| Тип | Появляющиеся поля |
|---|---|
| **Оригинальный** | новая АФС / известная АФС |
| **Воспроизведённый (generic)** | однокомпонентный/многокомпонентный; оригинальный препарат (наименование/дозировка/ЛФ/держатель/дата/№); **референтный препарат в исследованиях эквивалентности**; обоснование выбора референтного; рекомендации Экспертного комитета |
| **Биоподобный (биоаналог)** | оригинальный биологический; референтный биологический; различия с референтным |
| **Гибридный** | оригинальный препарат + отличия |
| **Травяной / гомеопатический / well-established / орфанный / radiopharm / из крови** | свои подразделы и условия |

### Форма II (перерегистрация) — отличия
Заголовок «о перерегистрации»; акцент на актуализацию данных, переоценку «польза–риск», ссылки на действующее РУ. Состав короче (привязка к модулям 1–2, см. B2/B5).

### Форма III (внесение изменений) — отличия
Заголовок «о внесении изменений в регистрационное досье»; ключевой блок **«2. Вносимые изменения»**: тип/класс изменения (IA/IB/II), затрагиваемые разделы, **было → стало**, обоснование. Остальные разделы заполняются по необходимости (в зависимости от типа препарата).

## МИ — формы заявлений (Решение 46)
- **Приложение № 2** — заявление о проведении **экспертизы**.
- **Приложение № 3** — заявление о проведении **регистрации**.
- Ключевые поля (из процедуры): наименование МИ, модели/марки, **класс потенциального риска** (Реш. 173), **вид по номенклатуре** (Реш. 177), область применения (Реш. 62), производитель + уполномоченный представитель, производственные площадки, наличие ЛС в составе, стерильность, наличие ПО, измерительная функция, реф. государство + государства признания.

## Условная логика (сводно)
| Управляющее поле | Влияние на форму |
|---|---|
| процедура (1.1.1/1.1.2/1.1.3) | блок реф./признания, идентификаторы |
| тип препарата (ЛС) | блок референтного/оригинального, биоаналог, эквивалентность |
| новая/известная АФС | мастер-файл АФС (досье) |
| класс риска (МИ) | объём досье/СМК/клиника |
| стерильность / ПО / ЛС в составе / измерит. функция (МИ) | доп. разделы экспертизы (B7) |

## Связь с реализацией (для C1)
В MVP 81 поле (A10) во многом покрывает Форму I (регистрация ЛС). **Не покрыто/частично:** специфические блоки Формы II (перерег) и Формы III («Вносимые изменения» — поля variation-* есть, но без полноценной формы/правил), а также поля Формы заявления МИ (класс риска как драйвер). Детальная сверка поле-в-поле — gap C1.
