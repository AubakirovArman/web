# B2. Процедуры ЛС (Решение Совета ЕЭК № 78)

Структура Правил (разделы): I Общие положения · II Определения · III Общие принципы регистрации · IV Общие принципы экспертизы · V Взаимное признание · VI Децентрализованная процедура · VII Пострегистрационные меры · VIII Подтверждение регистрации (перерегистрация) · IX Внесение изменений · X Приостановка/отзыв/ограничение · XI Условия обращения · XIII Приведение в соответствие · XIV Переходные · XV Особые случаи.

## Процедура РЕГИСТРАЦИИ
Два пути:
- **V. Процедура взаимного признания** — регистрация в референтном государстве, затем признание в государствах признания.
- **VI. Децентрализованная процедура** — одновременная подача в реф. государство + государства признания.

**Этапы (общие, разд. III–IV):**
1. Заявитель выбирает реф. государство + ≥1 государство признания.
2. Подаёт заявление + регистрационное досье (CTD) + оплату (+ образцы/стандарты при необходимости).
3. **Начальная экспертиза** — проверка комплектности и оформления (зона MVP).
4. **Специализированная экспертиза** — качество/безопасность/эффективность (модули 3/4/5).
5. Инспекция GMP (при необходимости).
6. Замечания эксперта (запрос) → ответ заявителя.
7. Экспертный отчёт по оценке → решение о регистрации → РУ.

## VIII. ПОДТВЕРЖДЕНИЕ РЕГИСТРАЦИИ (ПЕРЕРЕГИСТРАЦИЯ) — п.121–146
- Основание: **переоценка соотношения «польза–риск»** реф. государством.
- Бессрочное РУ после первой перерегистрации (если нет оснований для повторной).
- **Сроки:** процедура ≤ 80 раб. дней (реф.) / 50 раб. дней (признание). Заявление — **не ранее 260 раб. дней до истечения** срока РУ.
- Подаётся **во все государства**, где зарегистрирован препарат.
- **Документы (п.127):** (а) заявление на перерегистрацию (форма прил. № 20); (б) оплата; (в) **модули 1 и 2** регистрационного досье (актуализированные); экспертный отчёт реф. государства.
- Если держатель не подал заявление до истечения срока — РУ прекращает действие (п.124).

## IX. ВНЕСЕНИЕ ИЗМЕНЕНИЙ — п.147–155+
- Изменения вносятся по **классификации изменений** (Приложение № 19): **типы IA, IB, II** + административные.
  - **IA** — малые (уведомительный порядок, дополнение V прил. 19).
  - **IB** — малые, требующие оценки.
  - **II** — существенные → экспертиза, актуализация экспертного отчёта, **новое РУ**.
- Изменения не должны менять положительное «польза–риск» (п.151).
- Держатель обязан сообщать о новых сведениях, запретах/ограничениях (п.148–150).
- При одобрении — выдаётся новое РУ (п.153).

## XIII. Приведение в соответствие с правом Союза
Перевод национального досье в формат ЕАЭС (для ранее зарегистрированных по нац. правилам).

## Что влияет на состав/проверки (drivers процедуры)
- **Процедура** (регистрация / перерег / изменения) → разный пакет: регистрация = полное CTD; перерег = модули 1–2 + переоценка; изменения = по типу (IA/IB/II) + затрагиваемые разделы.
- **Тип препарата** (см. B4/B8) → объём модулей 4/5, биоэквивалентность, мастер-файл АФС и т.д.

## Связь с реализацией (для C4)
В MVP реализована только **регистрация** (правила документов). Перерегистрация и изменения как процедуры с собственным пакетом документов — **отсутствуют** (хотя поля вариаций и словарь `ls_variation_checklist_rules` есть).
